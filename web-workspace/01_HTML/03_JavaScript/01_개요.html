웹상에서 가장많이 사용하는 스크립트 언어!!
현시대 웹에서 화면을 만들어내는 기반기술로 사요되는것이 가장 큰 특징
(Angular, Vue, React)
네이버,카카오 -> Vue 를 선호한다 우리나라는 대부분 React를 선호한다 - 컴파일 : 컴파일러가 컴파일 단계를 거쳐서 실행
- 인터프리터 : 필요할 때 마다 한줄 한줄 바로 실행
- JIT 컴파일러 => 자주 실행되는 "핫 코드"를 기계어로 재 컴파일이라고 한다

자바 / 자바스크립트 => 컴파일 + 인터프리터

  • 자바스크립트에서 함수를 일급객체로 취급한다
  • 문법이 쉽다
    접근제한자 개념이 없다, 함수 반환형 없음, 변수 타입을 지정하지 않는다(타입추론)
  • 자바스크립트가 계속 발전하고 피드백이 빠름, 확장성이 좋아서 배워두면 여기저기에 사용가능하다

자바스크립트 소스코드 작성 위치에 따른 종류

  1. inline : 요소 속성에 간단한 자바스크립트 코드를 작성
  2. internal : HTML문서 내에 script태그에 코드를 작성
  3. external :코드를 별도의 .js파일로 작성해서 읽어오는 방법
    (모듈로 만들어서 export하고 쓰는 쪽에서 import해서 불러오는 방법도 있음)

1) inline 방식

        여는 태그릐 이벤트 속성에 간단한 소스코드를 작성하는 방식
        작성할 코드가 매우~~~~ 소량이거나
        다른 방식과 혼합할 경우 고려해볼 수 있다

2) internal 방식

        HTML문서에 script라는 태그 영역을 만든 뒤
        함수(function)단위로 소스코드를 작성하고
        특정 요소에 이벤트가 발생했을 때 해당 함수를 호출하는 방식이다
